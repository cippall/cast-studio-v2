# Product

## Register

product

## Users

Three distinct user types share the same interface with different feature sets:

- **Artists** — Create and manage actors, looks, and fashion items. Use AI generation to iterate on designs. Submit work to marketplace for client purchase. Primary workflow: creation → iteration → delivery.
- **Clients** — Browse marketplace, purchase actor packages and looks with credits. Commission custom work. Primary workflow: discover → buy → request.
- **Admins** — Manage models, taxonomy, users, marketplace listings, and commissions. Configure system-wide settings. Primary workflow: configure → review → approve.

## Product Purpose

Cast Studio is a multi-tenant digital casting platform that lets Artists use AI to generate hyper-realistic model actors, fashion looks, and wardrobe items. Clients purchase these assets on a credit-based marketplace. The product exists to remove the friction from digital casting — replacing physical photoshoots and wardrobe sourcing with an AI-powered creation pipeline.

## Brand Personality

editorial, precise, unhurried

## Anti-references

- **Not a dashboard** — Avoid dense widget grids, chart panels, KPI tiles, and the "SaaS control panel" feel. Data should breathe, not be crammed into a 12-column widget layout.
- **Not e-commerce** — Avoid product-card-with-buy-button grids, price-forward design, "Add to Cart" patterns, and promotional/sale UI. The marketplace is a gallery, not a storefront.
- **Not a creative tool panel** — Avoid Photoshop/Figma-style dense toolbars, floating property panels, and generation controls that dominate the screen. Controls should recede; results should be the focus.

## Design Principles

1. **Content is the interface** — The generated images ARE the product. Every pixel of chrome exists to frame them, never compete with them.
2. **Restraint is respect** — Sharp edges, no shadows, no rounded corners, no decorative elements. The design system trusts the user to be intelligent.
3. **Density matches purpose** — Library and marketplace pages that display many items use grid layouts. Detail and creation pages use generous whitespace. The layout serves the content, not a template.
4. **Generation is theater** — The async generation process should feel deliberate, not like a loading spinner. Progress states are moments of anticipation, not friction.

## Accessibility & Inclusion

- WCAG AA minimum for all interactive elements
- Focus rings are always visible (2px offset, high contrast)
- Status conveyed by color + text/icon, never color alone
- Reduced motion respected — all transitions degrade to instant or crossfade
